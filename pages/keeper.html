Keeper

Encrypted secret store. Reference secrets in config as ss://namespace/key

🗄️

Keeper is not configured on this node.
Add a security { keeper { ... } } block to your global config to enable encrypted secret storage. 

security {
  enabled = "on"
  keeper {
    enabled    = "on"
    passphrase = env.KEEPER_PASS   # or ss://… once bootstrapped
    auto_lock  = "30m"
  }
}
🔐

Enter your passphrase to unlock the secret store.
Secrets referenced as ss://namespace/path cannot be resolved until unlocked.
The store auto-locks after the timeout configured in security.keeper.auto_lock (default 30 min). While unlocked, the passphrase is held only in browser memory — never written to disk or storage.
