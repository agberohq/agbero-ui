Keeper

Encrypted secret store — reference as ss://namespace/key

🗄️ Keeper not configured Add a keeper block inside security { } in your global config to enable encrypted secret storage. 
security {
  keeper {
    enabled    = "on"
    passphrase = env.KEEPER_PASS
    auto_lock  = "30m"
    audit      = "on"
  }
}
🔐
Keeper is locked
Secrets referenced as ss://namespace/key cannot be resolved until the keeper is unlocked. The keeper auto-locks after the timeout set in security.keeper.auto_lock.
To unlock, restart agbero with the correct passphrase set in the environment, or use the CLI.
